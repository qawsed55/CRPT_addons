# 🛠️ Генератор УПД/СЧФДОП для Честного Знака

> ⚠️ **Важно**: Это неофициальный инструмент. Не имеет отношения к ООО «Оператор-ЦРПТ», системе «Честный ЗНАК» и ФНС России. Используйте на свой страх и риск.

## 📦 Что это?

Простой браузерный генератор XML-файлов формата **ОН_НСЧФДОП** (УПД/СЧФДОП) для передачи сведений о маркированных товарах в систему «Честный ЗНАК».

### Возможности:
- ✅ Генерация валидного XML по формату ФНС (версия 5.03)
- ✅ Поддержка маркировки: КИЗ, НомУпак, ИдентТрансУпак
- ✅ Таможенные данные: `СвДТ` (код страны, номер ДТ), `КрНаимСтрПр`
- ✅ Загрузка реквизитов из Excel-шаблона (многоразовое использование)
- ✅ Экспорт данных в CSV (с опцией построчной выгрузки КИЗ)
- ✅ Кодировка **Windows-1251** (реальное преобразование, не декларация)
- ✅ Валидация структуры по [XSD-схеме](./schemas/)
- ✅ Может просто сформировать CSV для импорта товаров в УПД через вэб-морду ЧЗ

## 📋 Требования к входным данным

### 1. Файл реквизитов (`requisites.xlsx`)
Формат: 3 столбца — `Field`, `Seller`, `Buyer`
> **EDOCode** — код оператора ЭДО (3 символа, например `2LT`). При работе без оператора — `000`.  
> **ParticipantID** — уникальный идентификатор участника (до 43 символов).

### 2. Файл товаров (`products.xlsx`)

| Столбец | Описание |
|---------|----------|

> 💡 Все денежные значения — с точкой как десятичным разделителем, 2 знака после запятой.

### 3. Файл маркировки (`marks.xlsx`)

> 🔹 Один GTIN может иметь несколько КИЗ — просто добавьте несколько строк.

## 🚀 Как использовать

1. Скачайте `generator.html` из репозитория
2. Откройте в браузере (Chrome / Яндекс.Браузере / Edge)
3. Заполните номер и дату документа
4. Загрузите 3 Excel-файла: реквизиты, товары, маркировка
5. Нажмите **«Сформировать XML»**
6. Проверьте результат в текстовом поле
7. Нажмите **«Скачать XML»** — файл будет в кодировке Windows-1251 с правильным именем
8. *(Опционально)* Нажмите **«ЭКСПОРТ CSV»** для выгрузки в табличном формате

## 📐 Формат выходного XML

- **Кодировка**: `windows-1251` (реальные байты, не декларация)
- **Имя файла**: совпадает с атрибутом `ИдФайл` в корне `<Файл>`
- **Структура**: полностью соответствует [Приказу ФНС России от 19.12.2023 № ЕД-7-26/970@](https://www.nalog.gov.ru/rn77/about_fts/docs/14414412/)
- **Версия формата**: `5.03`

### Пример имени файла:
```
ON_NSCHFDOPPR_2LT-3535353535_2LT-121212121_20251231_1428a1b2c3d4-..._0_1_0_0_0_00.xml
```

## 🔧 Разработка / доработка

- Генератор написан на чистом HTML + JavaScript, без внешних зависимостей (кроме `xlsx.full.min.js` для чтения Excel)
- Все вычисления происходят в браузере — данные не отправляются на сервер
- Не имеет деструктивных функций
